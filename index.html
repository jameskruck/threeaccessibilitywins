❌ BAD EXAMPLE

Inaccessible Data Table

This table has several accessibility problems:

  • No proper table headers (using regular cells instead of <th> elements)
  • Merged cells without proper scope attributes
  • No table caption
  • Screen readers cannot navigate this table properly
Department Q1 Sales Data Total
January February March
Marketing $45,000 $52,000 $48,000 $145,000
Sales $67,000 $71,000 $69,000 $207,000
Support $23,000 $28,000 $31,000 $82,000

Try using a screen reader on this table - notice how difficult it is to understand the relationships between headers and data cells.

✅ GOOD EXAMPLE

Accessible Data Table

This table fixes all the accessibility problems:

  • Proper <th> elements with scope attributes
  • Descriptive table caption
  • Clear header relationships
  • Screen readers can navigate effectively
Q1 2024 Sales Data by Department
Department Monthly Sales Q1 Total
January February March
Marketing $45,000 $52,000 $48,000 $145,000
Sales $67,000 $71,000 $69,000 $207,000
Support $23,000 $28,000 $31,000 $82,000

Screen reader users can now:

  • Understand what the table contains (caption)
  • Navigate between headers and data cells
  • Hear the appropriate headers when focused on any data cell
